## ¿Qué es SDD?

**Specification-Driven Development (SDD)** es una metodología de desarrollo que antepone la especificación explícita al código. En lugar de generar código directamente desde descripciones vagas, SDD establece un contrato claro entre el desarrollador y la IA antes de escribir una sola línea.

## SDD vs. Vibe Coding

| | Vibe Coding | SDD |
|--|-------------|-----|
| **Punto de inicio** | "Hazme una app de..." | Especificación detallada |
| **Iteraciones** | Prueba y error reactivo | Planificación → Implementación |
| **Comprensión del código** | Baja (copy-paste de IA) | Alta (el dev diseña, la IA ejecuta) |
| **Mantenibilidad** | Difícil | Predecible |
| **Control** | Mínimo | Total |
| **Resultado** | Código que "funciona" | Código correcto por diseño |

## El problema del vibe coding

El vibe coding produce código que funciona en el momento pero que:

- Nadie entiende del todo (ni el dev, ni la IA en la próxima sesión)
- Es difícil de depurar cuando falla
- Acumula deuda técnica de manera invisible
- No escala con el equipo

## SDD como solución

SDD convierte a la IA en un **ejecutor de alta precisión** de decisiones humanas, no en un tomador de decisiones autónomo. El desarrollador:

1. Define QUÉ debe hacer el sistema
2. Especifica CÓMO debe comportarse
3. Establece restricciones y criterios de éxito
4. Delega la implementación con contexto completo

El resultado es código que el equipo puede mantener, entender y extender.
