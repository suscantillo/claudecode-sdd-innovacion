title: Errores Comunes
description: Anti-patrones del copy-paste y del mal uso de Claude Code que debes evitar.
---

## Anti-patrón 1: El copy-paste ciego

**El problema**: Copiar código generado sin entenderlo, pegarlo en producción y descubrir el bug 3 días después.

**Señales de alerta:**
- "Funciona, no sé por qué"
- Código con nombres de variables genéricos (`data`, `result`, `temp`)
- Imports de librerías que no reconoces en tu proyecto

**La solución**: Si no entiendes el código generado, pide explicación antes de usarlo:
```
Antes de implementar esto, explícame qué hace cada parte
y por qué elegiste este enfoque.
```

## Anti-patrón 2: El prompt de una línea para tareas complejas

**El problema**: Pedir cosas complejas con contexto mínimo y esperar resultados perfectos.

```
# Esto no funciona bien
"Optimiza la base de datos"
"Arregla los tests"
"Mejora el rendimiento"
```

**La solución**: Especifica qué, dónde, cómo y con qué restricciones. Ver [patrón efectivo de trabajo](/workflow/patron-efectivo).

---

## Anti-patrón 3: Aprobar sin revisar

**El problema**: Hacer click en "Aprobar" en cada acción de Claude sin leer qué va a ejecutar.

Esto es especialmente peligroso con:
- Comandos `Bash` que borran o modifican archivos
- `git push` a ramas importantes
- Migraciones de base de datos

**La solución**: Lee siempre el contenido de la acción antes de aprobar. Si algo te parece extraño, pregunta.

---

## Anti-patrón 4: Sesiones maratónicas sin commits

**El problema**: Dejar que Claude haga 50 cambios en una sesión sin hacer commit intermedio. Si algo sale mal, la reversión es dolorosa.

**La solución**: Commit frecuente después de cada feature o corrección verificada:
```
# Al final de cada paso funcional:
Crea un commit con los cambios de [esta parte específica].
```

## Anti-patrón 5: Ignorar las advertencias de Claude

**El problema**: Cuando Claude dice "esto podría tener efectos secundarios en..." y el desarrollador responde "no importa, hazlo igual".

Claude advierte cuando detecta riesgos reales. Ignorar estas advertencias sin entenderlas lleva a bugs difíciles de rastrear.

**La solución**: Antes de ignorar una advertencia, pide que explique el riesgo:
```
Explícame el efecto secundario que mencionas. Si lo entiendo
y decido asumirlo, te lo confirmaré explícitamente.
```

## Anti-patrón 6: CLAUDE.md genérico

**El problema**: Un `CLAUDE.md` lleno de instrucciones obvias que no aportan valor específico al proyecto.

```markdown
# Mal CLAUDE.md
- Write clean code
- Add error handling
- Follow best practices
- Test your code
```

Esto consume ventana de contexto sin dar información útil.

**La solución**: Solo incluir lo que Claude no puede inferir del código mismo — convenciones no estándar, restricciones específicas del negocio, decisiones arquitectónicas no obvias.

---

## Anti-patrón 7: El prompt-párrafo sin spec

**El problema**: Pedir sistemas complejos en un solo mensaje sin documentar los requisitos:

```
"Crea un sistema de autenticación completo con login, registro,
recuperación de contraseña, OAuth con Google, manejo de sesiones..."
```

El modelo produce algo, pero sin saber tu stack ni tus constraints. El resultado requiere más tiempo de corrección que hacerlo desde cero.

**La solución**: Divide en fases con spec por cada feature. Empieza por el login básico.

---

## Anti-patrón 8: Ignorar que el CLAUDE.md puede ser sobreescrito

**El problema**: Asumir que el `CLAUDE.md` bloquea todas las instrucciones contrarias. Como demostró el [Caso 1](/casos-de-uso/caso-1-claude-md), Claude acepta instrucciones explícitas del usuario aunque contradigan el `CLAUDE.md`. Si le dices "no hagas X", lo omite — aunque X esté en las reglas.

**La solución**: Usa el `CLAUDE.md` como criterio de auditoría después de implementar:
```
"Revisa el CLAUDE.md. ¿Cumpliste todas las reglas en este endpoint?"
```

---

## Anti-patrón 9: No usar Plan Mode para refactors grandes

**El problema**: Lanzar un prompt de refactor grande en modo default puede resultar en cambios en 20 archivos difíciles de revertir mentalmente, aunque `/rewind` te salve técnicamente.

**La solución**: Usa `claude --permission-mode plan` para cualquier tarea que toque más de 5 archivos o código que no conoces bien.

---

## Anti-patrón 10: Demasiados servidores MCP conectados

**El problema**: Los servidores MCP cargan todas sus definiciones al inicio de cada sesión y consumen tokens de contexto. Tener 5 servidores conectados cuando solo usas uno es contexto desperdiciado.

**La solución**: Conecta solo los que estés usando activamente. Usa `/mcp` para ver el costo de cada servidor.
