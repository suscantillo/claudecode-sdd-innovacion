## Modos de operación

Claude Code puede operar en diferentes modos según el nivel de autonomía requerido:

| Modo | Comportamiento |
|------|---------------|
| **Por defecto** | Claude pide permiso antes de editar archivos y correr comandos |
| **Auto-accept edits** | Edita archivos sin preguntar, sigue pidiendo permiso para comandos |
| **Plan Mode** | Solo lectura — genera un plan que el usuario aprueba antes de ejecutar |

### Modo interactivo (por defecto)

Claude solicita confirmación antes de ejecutar acciones con impacto significativo (borrar archivos, push a repositorios, etc.).

### Modo autónomo

```bash
claude --dangerously-skip-permissions
```

:::caution
Usar solo en entornos controlados como pipelines CI/CD. Claude ejecutará todas las acciones sin pedir confirmación.
:::

### Plan Mode

El modo de planificación permite que Claude diseñe una estrategia antes de ejecutar cambios. Es especialmente útil para tareas complejas o código que no se conoce bien.

**Cuándo usar Plan Mode:**
- Tareas que afectan múltiples archivos
- Cambios arquitectónicos
- Cuando hay múltiples enfoques válidos
- Antes de refactorizaciones grandes

```bash
claude --permission-mode plan   # Iniciar directamente en Plan Mode
```

O desde dentro de una sesión activa:

```bash
/plan
```

Dentro de Plan Mode, usa `Ctrl+G` para abrir el plan en el editor de texto y editarlo antes de que Claude proceda.

**Configurar Plan Mode como comportamiento por defecto:**

```json
// .claude/settings.json
{
  "permissions": {
    "defaultMode": "plan"
  }
}
```

## Permisos por herramienta

El usuario puede aprobar o denegar herramientas individualmente. Las herramientas se agrupan en:

| Categoría | Herramientas | Riesgo |
|-----------|-------------|--------|
| Lectura | `Read`, `Glob`, `Grep` | Bajo |
| Escritura local | `Write`, `Edit` | Medio |
| Ejecución | `Bash` | Variable |
| Red | `WebFetch`, `WebSearch` | Bajo |
| Agentes | `Agent` | Variable |

## Checkpoints

Antes de cada edición de archivo, Claude hace un snapshot automático. Los checkpoints son **locales a la sesión, independientes de git, y persisten entre sesiones**.

```bash
Esc × 2  o  /rewind   # Abrir el menú de checkpoints
/checkpoints           # Ver historial de checkpoints
```

Desde el menú de checkpoints puedes restaurar archivos, la conversación, o ambos.

## Hooks de permisos

Los hooks son comandos de shell que se ejecutan en respuesta a eventos de Claude Code, permitiendo control programático de los permisos:

```json
// .claude/settings.json
{
  "hooks": {
    "PostFileEdit": ["npx eslint --fix $FILE"],
    "PreToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "echo 'Bash ejecutándose'" }] }]
  }
}
```

Eventos disponibles: `PostFileEdit`, `PreToolUse`, `PostToolUse`, `Notification`, `WorktreeCreate`, `WorktreeRemove`.
